Microsoft 365 SMTP Relay
Envio de e-mails sem autenticação por sistemas internos

A única forma suportada pela Microsoft para envio de e-mails sem usuário e senha em aplicações legadas.

O problema dos sistemas internos com SMTP moderno

Sistemas internos, ERPs, softwares de cartório, aplicações legadas, impressoras e dispositivos de automação frequentemente não suportam autenticação moderna (OAuth) exigida pelo Microsoft 365.

Tentativas comuns incluem configurar SMTP com usuário e senha, desabilitar TLS ou reutilizar credenciais administrativas — práticas que resultam em falhas ou bloqueios.

O que NÃO é permitido no Microsoft 365

  • envio sem autenticação usando smtp.office365.com
  • uso de usuário e senha sem TLS
  • relays abertos sem validação de origem

Essas abordagens violam as políticas de segurança do Microsoft 365 e não são suportadas.

A solução correta: SMTP Relay com conector

O SMTP Relay permite que sistemas enviem e-mails sem autenticação, utilizando validação por:

  • endereço IP público
  • domínio autorizado
  • infraestrutura segura do Exchange Online

Pré-requisitos

  • domínio no Microsoft 365 (ex: suaempresa.com.br)
  • IP público fixo do local de envio
  • acesso ao Exchange Admin Center
  • controle do DNS do domínio

Configuração do DNS (obrigatória)

Registro MX - Bem provável já existe não precisa alterar, caso não exista crie a entrada.

suaempresa-com-br.mail.protection.outlook.com

Registro SPF - Já deve existir irá apenas adicionar o IP fixo de origem como válido para envio de emails em nome do seu dominio.

v=spf1 ip4:SEU.IP.PUBLICO include:spf.protection.outlook.com -all

Atenção:
Deve existir apenas um registro SPF no domínio. Múltiplos SPFs causam falhas de entrega.

Configuração no sistema interno

Parâmetro Valor
Servidor SMTP suaempresa-com-br.mail.protection.outlook.com
Porta 25
Usuário (vazio)
Senha (vazio)
SSL / TLS Desativado

Remetente (From)

O endereço de envio deve existir no tenant do Microsoft 365, mas não precisa ter licença:

notificacoes@suaempresa.com.br

Limitações e boas práticas

  • limite aproximado de 10.000 mensagens/dia por IP
  • não indicado para campanhas de marketing
  • porta 25 deve estar liberada no firewall
  • SPF incorreto resulta em spam ou bloqueio

Segurança adicional recomendada

  • ativar DKIM no Microsoft 365
  • implementar DMARC em modo monitoramento

SMTP Relay é a única forma suportada pela Microsoft para envio de e-mails sem autenticação.
