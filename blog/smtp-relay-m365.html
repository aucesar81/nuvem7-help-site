Microsoft 365 SMTP Relay
Envio de e-mails sem autenticação por sistemas internos

A única forma suportada pela Microsoft para envio de e-mails sem usuário e senha em aplicações legadas.

Falar com Especialista

O problema dos sistemas internos com SMTP moderno

Sistemas internos, ERPs, softwares de cartório, aplicações legadas, impressoras e dispositivos de automação frequentemente não suportam autenticação moderna (OAuth) exigida pelo Microsoft 365.

A solução correta: SMTP Relay com conector

O SMTP Relay permite envio de e-mails sem autenticação, utilizando validação por IP público e domínio autorizado.

Pré-requisitos

  • domínio no Microsoft 365
  • IP público fixo
  • acesso ao Exchange Admin Center
  • controle do DNS

Configuração de DNS

MX

suaempresa-com-br.mail.protection.outlook.com

SPF

v=spf1 ip4:SEU.IP.PUBLICO include:spf.protection.outlook.com -all
Atenção: deve existir apenas um registro SPF no domínio.

Configuração no sistema

Servidorsuaempresa-com-br.mail.protection.outlook.com
Porta25
Usuáriovazio
Senhavazio
SSL/TLSDesativado

Conclusão

Para sistemas que não suportam autenticação, o SMTP Relay é a única solução suportada e segura no Microsoft 365.
