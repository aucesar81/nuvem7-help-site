The Active Directory replication error RPC Server Unavailable (1722) is one of the most common and searched issues in Windows environments today.

Common symptoms

  • Login failures on specific workstations
  • Group Policy not applying
  • Users not replicating across all domain controllers
  • Replication errors in NTDS logs

Root cause of error 1722

This error indicates that one domain controller cannot communicate with another via RPC. It is usually caused by:

  • DNS misconfiguration (most common cause)
  • Firewall blocking dynamic RPC ports
  • Network connectivity issues
  • Time synchronization problems

Risks if not fixed

  • User and group inconsistencies
  • Intermittent authentication failures
  • Outdated Group Policies
  • Logical Active Directory corruption

Proper troubleshooting approach

  1. Validate DNS configuration
  2. Run dcdiag and repadmin /replsummary
  3. Test RPC connectivity
  4. Check time synchronization
