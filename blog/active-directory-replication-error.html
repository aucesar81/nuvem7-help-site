Erro de Replicação no Active Directory (RPC 1722)
Active Directory Replication Error (RPC Server Unavailable)

O erro de replicação no Active Directory, especialmente o RPC Server Unavailable (Erro 1722), é atualmente um dos problemas mais pesquisados por administradores Windows. Ele impacta diretamente autenticação, GPOs, criação de usuários e estabilidade do domínio.

The Active Directory replication error RPC Server Unavailable (1722) is one of the most common and searched issues in Windows environments, affecting authentication, Group Policies and overall domain health.

Principais sintomas | Common symptoms

  • Falha de login em estações específicas
  • GPO não aplica corretamente
  • Usuários criados que não aparecem em todos os DCs
  • Eventos de erro no log NTDS Replication
  • Replication errors reported by repadmin

Causa técnica do erro 1722 | Root cause

Esse erro indica que um controlador de domínio não consegue se comunicar com outro via RPC. Na prática, quase sempre envolve:

  • Problemas de DNS (causa nº 1)
  • Firewall bloqueando portas RPC dinâmicas
  • Conectividade de rede instável
  • Problemas de resolução de nomes entre DCs
  • Diferença de horário entre controladores

This error means one domain controller cannot establish RPC communication with another, usually caused by DNS misconfiguration, firewall restrictions or network issues.

Riscos de não corrigir | Risks if not fixed

  • Inconsistência de usuários e grupos
  • Falhas intermitentes de autenticação
  • GPOs desatualizadas
  • Risco de corrupção lógica do Active Directory
  • Impacto direto em sistemas críticos

Abordagem correta de diagnóstico | Proper troubleshooting approach

  1. Validar DNS (forward e reverse)
  2. Executar dcdiag e repadmin /replsummary
  3. Testar conectividade RPC entre DCs
  4. Validar sincronização de horário
  5. Revisar regras de firewall

A correção definitiva exige análise do ambiente como um todo, não apenas reiniciar serviços ou forçar replicação.
